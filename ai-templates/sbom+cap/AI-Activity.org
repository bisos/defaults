* About BISOS, Blee and ByStar

ByStar (By*) is a Libre-Halaal digital ecosystem — a non-proprietary
autonomy-oriented complete digital ecosystem.

BISOS (ByStar Internet Services Operating System) is the technical
foundation of ByStar. It's a layer on top of Debian that unifies
service-side and usage-side software into a single "software-service
continuum." Its basic unit is a *site* (private git server + registrar),
and its core abstractions are BPOs (portable git-based packages of
data/config/software) and PALS (Possession-Assertable Libre Services).

Blee (BISOS Libre-Halaal Emacs Environment) is the primary usage and
development environment for BISOS — a heavily-integrated Emacs layer.
Blee and BISOS are intertwined.

For concepts and terminology, import:
@/bisos/panels/bisos-core/modelAndTerminology/_nodeBase_/fullUsagePanel

* About bisos.sbom -- BISOS-SBOMs (Software Bill Of Materials)

Installment of debian packages, python pip and pipx packages, creation of
associated directories and cloning of git repos are all facilitated through the

** Web Virtual Domain Services (wvds) Pattern

This is a common category of web services in which a service run at a port and then
is exposed at a virtual domain. In addition to the ByStar domain specification is general
the web service is also exposed at localhost.

See

@/etc/hosts

As a wvds, consider airflow in the context of airflow.here of /etc/hosts as an example:

@/bisos/asc/web/bin/airflow-sbom.pcs                 :: Installs airflow and its dependencies
@/bisos/asc/web/bin/airflow-webserver-sysd.pcs       :: Creates systemd entry/ies for airflow
@/bisos/asc/web/bin/airflow-here-wvds-nginx.pcs      :: NOTYET, creates virdom entry for here and airflow
@/bisos/asc/web/bin/airflow-here-dns.pcs             :: NOTYET, for here, run dblock on etc/hosts
@/bisos/asc/web/bin/airflow-cbs-wvds-np-sysd.pcs     :: Does all of the above
