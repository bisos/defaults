#+title: Development Status & Continuation Handoff
#+TAGS: me(m) ai(a)

This file is a working-state handoff so development can continue on another
machine (which has this repo but NOT the developer's =~=). It captures the
architecture, what works, what is untested and the design decisions made so far.

This file is to be maintained by Claude. When a release is being made, claude should be
prompted to update this document.

Companion docs: [[file:AI-AGENTS.org][AI-AGENTS.org]] (agent guidance / layering), [[file:README.org][README.org]] (overview).

* Current State: Functional

=startAiAt.cs= is fully implemented and in use.

* Architecture

Single PyCS-XU script at =/bisos/apps/defaults/ai-templates/startAiAt.cs/startAiAt.cs=.
Templates base: =/bisos/apps/defaults/ai-templates/=.

One primary command: =initiate=.

Parameters:
- =activity= (mandatory): must match a subdirectory under the templates base
- =root= (optional, default =curDir=): =curDir= = cwd, =repo= = git repo root

** What =initiate= does

1. Symlinks =CLAUDE.md=, =AI-AGENTS.org=, =AI-WORKFLOW.org= from =bystar/=
2. Symlinks =AI-Activity.org= from =<activity>/=
3. Safe-copies =AI-DevStatus.org= and =AI-WorkPlan.org= from =<activity>/=,
   then runs =bx-dblock -i dblockUpdateFiles= on each to expand the
   =b:ai:file/particulars= dblock with target-directory context
4. Safe-copies =_claude/= contents into =.claude/= at the target
   (prefers activity-specific =_claude/= over =bystar/_claude/= as fallback)

All operations skip if destination already exists (safe/idempotent).

** File Naming Convention

All-caps = constant/symlinked (hands-off): =CLAUDE.md=, =AI-AGENTS.org=, =AI-WORKFLOW.org=
Mixed-case = local/editable: =AI-Activity.org=, =AI-DevStatus.org=, =AI-WorkPlan.org=

** _claude/ layout

Template source is =_claude/= (visible in repo); installed as =.claude/= in target.

=settings.json= pre-authorizes: =Read(*)=, =startAiAt.cs*=, =git= read-only
commands, =find=, =ls=, =cat=, =grep=, venv python, pyflakes, flake8.

Slash commands in =_claude/commands/=:
- =bx-confirm0= :: enumerate imported files and confirm understanding at session start
- =bx-status= :: summarize current dev state and list open TODOs
- =bx-done <anchor>= :: mark a TODO DONE and write AI Results section
- =bx-update-status= :: rewrite AI-DevStatus.org to reflect current state
- =initiate= :: (for non-Emacs workflows only) run =startAiAt.cs= then orient Claude;
  not relevant to Emacs workflow where CLAUDE.md is read automatically at startup

** examples command

Dynamically discovers activity directories by listing =g_templatesBase=, excluding
=bystar=, =startAiAt.cs=, and =test=. Generates two =menuChapter= sections:
- =root=curDir= (default) — one entry per activity
- =root=repo= — one entry per activity

New activities appear automatically without editing the script.

** b:ai:file/particulars dblock

Defined in =dblock-comeega-prog.el=. Emits into =AI-WorkPlan.org= and
=AI-DevStatus.org= at copy time:
- Working Directory
- File (canonical path)
- Activity (resolved from =AI-Activity.org= symlink target)
- Companion Docs (dynamically, based on files present in directory)

* What Is Untested

- =bx-dblock= failure path (non-zero return) — only logged, not fatal
- Activity-specific =_claude/= override (most activities use =bystar/_claude/=)
- =root=repo= mode across all activity types
